# 局域网炸金花

一个给 NAS / Docker 部署的轻量炸金花网页游戏。手机和电脑都可以打开浏览器一起玩，支持局域网访问，也可以配合内网穿透给外地朋友加入。

## 功能

- 房间号加入，同一个局域网内分享链接即可进房
- 可选房间密码，适合内网穿透或公网访问
- 2 到 6 人实时牌局
- 看牌、闷跟、明跟、加注、比牌、弃牌
- 自动洗牌、发牌、判型、结算、下一局
- 手机端适配，无需安装 App

## 本地运行

```bash
node server.js
```

打开 `http://localhost:3000`。

## Docker 部署到 NAS

在本目录执行：

```bash
docker compose up -d --build
```

然后让朋友在同一局域网访问：

```text
http://NAS的局域网IP:3000
```

例如：

```text
http://192.168.1.20:3000
```

## 内网穿透给朋友玩

先在 NAS 上按 Docker 方式启动本项目，再把 NAS 的 `3000` 端口通过你的内网穿透工具映射出去。

外部访问链路通常是：

```text
朋友浏览器 -> 你的穿透域名/公网地址 -> NAS:3000 -> 炸金花服务
```

建议这样使用：

1. 创建房间时填写房间密码。
2. 把穿透后的访问地址发给朋友，例如 `https://你的域名.example.com`。
3. 朋友打开页面，输入房间号、昵称、房间密码加入。
4. 如果穿透工具支持 HTTPS，优先使用 HTTPS。

不要把没有密码的房间长期暴露在公网。这个项目是朋友间娱乐用的轻量服务，没有账号系统、支付系统或真钱结算能力。

## 可配置项

可以在 `docker-compose.yml` 里改这些环境变量：

- `PORT`: 容器监听端口，默认 `3000`
- `STARTING_CHIPS`: 每位玩家初始筹码，默认 `1000`
- `ANTE`: 底注，默认 `10`
- `MAX_PLAYERS`: 房间最大人数，默认 `6`
- `MAX_FAILED_JOINS`: 同一来源 10 分钟内允许输错房间密码的次数，默认 `20`

## 当前规则

牌型大小为：

```text
豹子 > 同花顺 > 同花 > 顺子 > 对子 > 单张
```

本版本暂不启用各地差异较大的特殊规则，例如 `235 吃豹子`。看牌后跟注和加注按双倍支付；比牌需要先看牌。
